Ready-to-Use Review Prompts

Use these prompts after installing the secure-review skill in Claude Code or Codex.

Recommended sequence: run the master review prompt, run false-positive reduction, then generate the final report from validated findings only.

Prompt Set

Full Application Review

End-to-end detection-only review with attack surface, role, permission, trust-boundary, exploitability, and false-positive gates.

Maximum Discovery

Broad specialist-agent coverage while keeping final findings restricted to validated vulnerabilities.

False-Positive Reduction

Aggressively removes findings without reachability, attacker control, source-to-sink proof, impact, or effective evidence.

Access Control

Focuses on role mapping, permission mapping, IDOR/BOLA, tenant isolation, and admin boundary bypasses.

Master Prompt

Use the secure-review skill.

Review this application end to end in detection-only mode. Do not modify code unless I explicitly approve.

Your goal is to find real, reachable, exploitable, evidence-backed security vulnerabilities with the lowest possible false-positive rate.

Start with application profiling, technology stack detection, attack surface mapping, authentication flow mapping, role mapping, permission matrix creation, trust boundary mapping, data flow mapping, vulnerability playbook selection, deterministic scan review, specialist agent review, exploitability validation, false-positive validation, and final reporting.

Do not start reporting vulnerabilities until you complete attack surface, role, permission, and trust-boundary mapping.

Only include TRUE POSITIVE and LIKELY TRUE POSITIVE in the final vulnerability section. Move weak, theoretical, scanner-only, or uncertain issues to Needs Manual Validation / Hardening Notes.
