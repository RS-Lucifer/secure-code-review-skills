Skill Anatomy

Reference anatomy for an enterprise secure source-code-review skill with strong evidence gates and low false positives.

The current package is intentionally compact. This anatomy documents the target structure and review model for future expansion.

Core Flow

  1. Scope and rules
  2. Application profiling
  3. Technology detection
  4. Attack surface mapping
  5. Role and permission mapping
  6. Trust boundary mapping
  7. Vulnerability playbook selection
  8. Semgrep / SCA / secrets review
  9. Agent-based review
  10. Exploitability and false-positive validation
  11. Evidence hygiene, risk scoring, and report generation

Coverage

Authorization

Access control, IDOR/BOLA, tenant isolation, role bypass, admin boundary, and service-account permission issues.

Input and Data Flows

Injection, SSRF, file upload, path traversal, deserialization, mass assignment, and excessive data exposure.

Platform

Cloud, IAM, CI/CD, supply chain, SCA/SBOM, mobile API authorization, and secrets exposure.

Validation

Reachability, attacker control, source-to-sink proof, existing controls, realistic impact, and evidence quality.
