# Security Policy

## Supported Use

This project is for authorized defensive source code review.

Do not use the packages for destructive testing, production exploitation, credential use, or testing outside an approved scope.

## Reporting Issues

Open a GitHub issue for packaging bugs, broken validation scripts, documentation errors, or workflow problems.

Do not paste live secrets, tokens, private keys, cookies, customer data, or exploit output into public issues.

## Evidence Standard

Security claims should include reachable code evidence, affected entry points, attacker role, impacted object or tenant where relevant, realistic impact, and safe retest steps.
